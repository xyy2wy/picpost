# Semi-Utils 使用文档

本文档面向第一次接触本项目的用户，覆盖环境准备、命令行用法、前端用法、常见配置项和排错建议。

## 1. 项目简介

`semi-utils` 是一个批量图片处理工具，主要用于：

- 给照片添加底部水印信息
- 生成不同风格的版式
- 添加白边、阴影、背景模糊等效果
- 一键调色 / 滤镜、自动增强
- 平铺 / 单点防盗文字水印
- 按统一尺寸输出图片，或一键导出多个比例
- 长图切割 / 九宫格、拼图封面、首图文字卡片、页码角标等小红书多图玩法
- 批量处理 `input/` 目录中的图片

当前项目支持两种使用方式：

- 命令行菜单模式：适合沿用原有交互方式
- Web 前端模式：适合可视化调整配置、预览和下载结果

当前 Web 前端还支持：

- AI 预设助手：根据样图和目标描述自动推荐一套适合批量处理的参数
- AI 文案 + 话题标签：根据一组图片生成小红书标题/正文/标签（支持 OpenAI / Google Gemini）

## 2. 目录说明

## 3. 环境准备

### 3.1 推荐方式：使用 Miniconda

如果你已经按照当前仓库的配置安装了 `Miniconda`，推荐使用独立环境运行本项目。

激活环境：

```bash
source ~/.zshrc
conda activate semi-utils
cd /Users/xyy/Desktop/semi-utils
```

如果你还没有安装依赖，执行：

```bash
python -m pip install -r requirements.txt
```

### 3.2 依赖列表

当前项目核心依赖包括：

- `pillow`
- `pyyaml`
- `tqdm`
- `python-dateutil`
- `requests`
- `streamlit`
- `openai`（AI 预设 / 文案 / 选图 / 风格建议，兼容 OpenAI 接口，可对接 Gemini）

可选依赖：

- `opencv-python`：仅用于智能裁图的人脸优先检测。不安装时智能裁图自动回退纯 PIL 显著性裁切，不影响任何核心流程。

## 4. 支持的图片格式

目前会识别以下格式：

- `.jpg`
- `.jpeg`
- `.png`

建议优先使用带 EXIF 的 JPEG 照片，这样相机型号、镜头、拍摄参数、日期等字段会更完整。

## 5. 快速开始

### 5.1 命令行模式

1. 把需要处理的图片放入 `input/`
2. 进入项目目录并激活环境
3. 运行：

```bash
python main.py
```

4. 根据菜单调整配置
5. 输入 `y` 或直接回车开始处理
6. 结果会输出到 `output/`

### 5.2 Web 前端模式

1. 进入项目目录并激活环境
2. 启动前端：

```bash
streamlit run web_app.py
```

3. 浏览器打开本地地址，一般是：

```text
http://localhost:8501
```

如果端口被占用，`streamlit` 会自动提示新的端口，或者你也可以手动指定：

```bash
streamlit run web_app.py --server.port 8765
```

4. 在页面中选择：
   - 上传图片
   - 或直接处理 `input/` 文件夹
5. 设置布局和处理参数
6. 点击开始处理
7. 预览并下载结果

## 7. Web 前端详细说明

### 7.1 启动命令

```bash
streamlit run web_app.py
```

### 7.2 页面结构

前端主要分成两部分：

- 左侧边栏
  用于配置处理参数
- 主区域
  用于选择图片来源、执行处理、预览结果和下载

### 7.3 图片来源

前端支持两种输入方式：

#### 上传图片

适合临时处理少量图片。

特点：

- 支持一次上传多张
- 结果会写入 `output_web/`
- 页面可直接下载单张或整包 ZIP
- 可以直接调用 AI 预设助手，用第一张上传图片作为样图生成推荐参数

#### input 文件夹

适合你已经把图片整理到项目目录中的场景。

特点：

- 直接读取 `input/`
- 结果输出到 `output/`
- 保留原来的批处理工作方式
- 可使用 `input/` 中第一张图片作为 AI 样图

### 7.3 AI 预设助手

Web 前端中的 AI 预设助手可以根据一张样图和一句目标描述，自动推荐一套参数。

你需要提供：

- 服务商（OpenAI / Google Gemini / 自定义）
- API Key
- 模型（下拉选择，随服务商联动）
- 可选 Base URL
- 目标描述

服务商与模型选择：

- **OpenAI**：`gpt-5`（综合最强、较慢）、`gpt-5-mini`（更快更省）、`gpt-4.1` / `gpt-4.1-mini`、`gpt-4o`（多模态成熟）、`gpt-4o-mini`（最省，适合大批量）。
- **Google Gemini**：`gemini-2.5-pro`（最强推理）、`gemini-2.5-flash`（均衡快速）、`gemini-2.5-flash-lite`（最省最快）、`gemini-2.0-flash`。选中后会自动使用 Gemini 的 OpenAI 兼容端点，一般无需手填 Base URL。
- **自定义**：用于任意兼容 OpenAI 接口的第三方/自建服务，需自行填写 Base URL 与模型名。
- 任何服务商下都可选「自定义」手动填写模型名（如指定快照版本 `gpt-4o-2024-11-20`）。
- 切换服务商时会自动把模型重置为该服务商的首选模型，并尝试用对应环境变量（`OPENAI_API_KEY` / `GEMINI_API_KEY`）自动填充 Key。
- AI 预设助手与 AI 文案共用同一处服务商/模型/Key/Base URL 设置。

底层统一走 OpenAI 的 Chat Completions 接口（OpenAI 与 Gemini 兼容端点的共同子集），
并优先用 JSON 模式强制结构化输出；发送前图片会自动下采样，降低上传体积与成本。
界面还会给出本次请求的输入 token 粗略预估，便于控制花费。

如果你已经在终端中设置了环境变量，也可以直接复用：

```bash
export OPENAI_API_KEY=你的密钥     # OpenAI
export GEMINI_API_KEY=你的密钥     # Google Gemini
```

推荐用法：

1. 先选择图片来源
2. 打开 `AI 预设助手`
3. 选择服务商与模型，填入 Key
4. 输入目标描述
5. 点击生成建议
6. 查看 AI 推荐参数是否符合预期
7. 再批量处理整组图片

示例目标：

- 我要发布到小红书，画面干净统一，保留拍摄参数
- 我要做一组极简白边作品，偏摄影作品集风格
- 我要批量导出竖版封面图，要求统一尺寸，少一点花哨元素

### 7.4 AI 文案 + 话题标签

在「小红书工具 → AI 文案标签」中，上传一组要发布的图片，AI 会综合整组照片
生成小红书风格的标题、正文和话题标签：

- 与预设助手共用同一套服务商/模型/Key 设置。
- 可设置「生成候选数量」（1-5），一次产出多套标题/正文，分标签页对比挑选，话题标签整组共用。
- 每套候选都能单独下载为 `.txt`。

### 7.5 调色 / 滤镜 / 自动增强

在侧边栏「调色 / 滤镜」中：

- 内置滤镜预设：清新、胶片、暖调、冷调、Ins 风、黑白。
- 高级模式下可手动微调亮度、对比度、饱和度、锐化、色温。
- 「自动对比度增强」按直方图自动优化明暗。
- 调色在整个处理链最前面执行（先调色，再排版/加水印），命令行「更多设置」里也有同样的选项。

### 7.6 防盗文字水印

在侧边栏「防盗水印」中：

- 填写水印文字（如 `@你的昵称`），可选整图斜向平铺或单点放置。
- 可调不透明度。水印在处理链最后打在最终成图上。

### 7.7 首图文字卡片 / 多比例导出（小红书工具）

- 首图卡片：在纯色或图片背景上生成带大标题、副标题的封面图，适合做轮播第一张；标题会自动换行。
- 多比例导出：把每张图同时导出成 3:4 / 1:1 / 9:16 / 4:3 等多个比例，发不同位置直接用。

### 7.8 批量选片与 AI 智能选图（小红书工具）

在「小红书工具 → 选片」中，可以从一组照片里快速圈定要发布的几张：

- 以网格缩略图展示全部图片，缩略图会自动下采样，浏览大批量照片也不卡。
- 点击图片切换「选中 / 未选中」，并支持 1–5 星打分（0 表示未评分）。
- 支持「仅看已选中」或「按星级过滤」，只展示满足条件的图片。
- 点击「导出选中」把选中的图片按当前顺序复制到目标目录，文件名可选加 `01_`、`02_` 序号前缀。
- 单张图片读取失败会自动跳过，不会中断整个选片流程。

「AI 选图」按钮可在选片基础上调用视觉模型：

- 提供目标张数 N 和（可选）发布目标描述，AI 返回推荐子集、推荐顺序与简短理由，并标注到选片状态上。
- 发送前会下采样并限制单次图片数量，控制 token 成本，界面给出输入规模/成本粗估。
- 调用失败或返回无效项时安全降级，保留你已有的手动选片结果。
- 未安装 `openai` 或未配置 Key 时只提示，不影响手动选片。

### 7.9 全维度风格包（保存 / 应用 / 导入导出）

在侧边栏「风格包」中，可以把一整套处理风格保存为命名风格包并一键复用：

- 快照覆盖全部维度：布局、Logo 开关与位置、四角文字、白边、阴影、统一尺寸、调色 / 滤镜、防盗水印、输出质量。
- 支持保存、应用、删除自定义风格包（内置风格包不可删除）。
- 「导出」把风格包下载为可分享的 JSON 文件；「导入」上传 JSON，校验结构后加入可用列表，非法文件只提示不污染现有数据。
- 风格包文件缺字段时用安全默认值补齐，保证应用不报错，并兼容已有的 `custom_presets.json` 历史数据。

### 7.10 实时预览

在主区域「实时预览」中，调整布局 / 调色 / 水印等任一参数后无需点「处理」即可看到样图效果：

- 预览基于下采样样图，速度快，与正式输出共用同一套处理链，视觉效果一致（仅分辨率不同）。
- 预览出错时保留上一次成功的预览并给出提示，不影响正式处理入口。
- 没有样图时会提示先上传或选择图片。

### 7.11 图上标注 / 文字贴纸（小红书工具）

在「小红书工具 → 标注」中，可以在图片上叠加文字标注，突出重点信息：

- 支持三种样式：气泡（圆角底 + 文字）、纯文字（描边）、价格标签（高亮底色）。
- 可设置文字内容、文字颜色、底色、字号与相对位置。
- 一张图可添加多个标注，按添加顺序全部渲染；文字为空的标注会跳过。
- 渲染返回新图，不修改原图。

### 7.12 长图拼接 / 前后对比（小红书工具）

在「小红书工具 → 长图/对比」中，可把多张图合成一张：

- 竖向长拼接：把多张图等宽对齐后纵向拼接为长图，图片间可选间距与背景色。
- 前后对比：把两张图缩放对齐后按左右或上下布局拼接，可选中缝分隔线与「Before/After」（或自定义）标签。
- 拼接返回新图，不修改原图；输入图片少于 2 张时会给出提示。

### 7.13 智能裁图（主体 / 人脸感知）

在统一尺寸或多比例导出裁切时，可选择「智能裁切」策略，让主体不被切掉：

- 统一尺寸模式新增「智能裁切」选项（除原有的留白补齐 / 居中裁切 / 强制拉伸外）。
- 默认基于纯 PIL 显著性（边缘能量重心）估计主体位置，无需任何额外依赖即可使用。
- 若额外安装了可选的 `opencv-python`，会优先做人脸检测，保证人脸不被裁掉。
- 检测不可用或失败时安全回退为居中裁切，绝不报错；裁切输出尺寸严格等于目标尺寸。

### 7.14 AI 风格 / 标签建议

在 AI 工具中，可让模型看图后直接给出风格与标签建议：

- 返回推荐的滤镜预设名（取值限定在现有滤镜集合内，超出则回退「原图 / none」）及理由。
- 返回去 `#` 的话题标签列表，复用现有文案标签归一化。
- 复用与 AI 预设助手相同的多服务商、下采样、错误处理与成本预估机制。
- 「采纳滤镜」可一键把建议的滤镜应用到当前配置。

### 7.15 封面单独样式（批次内多样式）

在工作台中开启「封面单独样式」后，可对封面与内页用不同风格：

- 指定封面图（默认第一张），封面应用风格包 A、其余图片应用风格包 B。
- 未开启时对全部图片应用同一套配置，保持原有行为。
- 指定的封面序号超出范围时自动回退为第一张。

### 7.16 发布草稿打包

处理完一组图并生成文案后，可在处理结果区点击「打包发布草稿」：

- 生成一个草稿（目录或 ZIP），包含按 `01_`、`02_` 顺序命名的图片，以及含标题 / 正文 / 标签的文本文件。
- 顺序前缀保证上传顺序与处理顺序一致。
- 没有文案时仍可只打包图片，文本文件为空或仅含占位。
- 输出至 `output_xiaohongshu/draft`，Web 提供下载、CLI 输出明确路径。

### 7.17 边栏参数说明

#### 布局

控制整体输出风格。

#### 启用 Logo / Logo 位置

控制是否显示 Logo，以及 Logo 在左还是在右。

#### 四个文字区域

可以分别设置：

- 左上角
- 左下角
- 右上角
- 右下角

如果选择“自定义”，页面会出现额外输入框。

#### 白色边框

控制是否给图像增加白边，以及白边宽度。

#### 阴影

给图像增加阴影感，通常适合非正方形布局。

#### 等效焦距

启用后，参数文字中的焦距会优先使用等效焦距。

#### 按原比例补边

用于在一些布局处理之后，尽量回到原始图像比例。

#### 统一尺寸

启用后，输出图片会统一到固定宽高。

可设置：

- 统一宽度
- 统一高度
- 统一尺寸模式

#### 输出质量

范围为 `50-100`。

值越高：

- 图片越清晰
- 体积通常越大

## 8. 统一尺寸功能说明

统一尺寸是新增的一个重要能力，用于把所有输出图统一到同一分辨率。

### 8.1 开关

位置：

- 命令行：`更多设置 -> 统一尺寸`
- 前端：边栏中的 `统一尺寸`

### 8.2 尺寸设置

可设置：

- 宽度
- 高度

默认值：

```text
1080 x 1350
```

### 8.3 三种模式

#### 留白补齐

逻辑：

- 保持原始比例缩放
- 不裁切内容
- 空余区域用背景色补齐

适合：

- 不希望裁掉画面
- 想保证构图完整

特点：

- 最稳妥
- 默认推荐模式

#### 居中裁切

逻辑：

- 先按比例放大
- 再从中心裁切到目标尺寸

适合：

- 更重视画面填满
- 接受边缘部分被裁掉

特点：

- 画面更满
- 容易裁掉上下或左右边缘内容

#### 强制拉伸

逻辑：

- 直接拉伸到目标宽高

适合：

- 明确需要固定尺寸
- 不在意轻微变形

特点：

- 一定能完整铺满
- 可能导致人物或物体变形

#### 智能裁切

逻辑：

- 估计主体 / 显著区域位置（默认纯 PIL 显著性；装了 `opencv-python` 时优先人脸检测）
- 以主体为锚点裁切到目标尺寸，尽量不切到主体 / 人脸

适合：

- 多比例导出且画面有明确主体 / 人物

特点：

- 主体更不易被裁掉
- 检测不可用或失败时自动回退为居中裁切，输出尺寸始终等于目标

## 9. 输出结果说明

### 9.1 命令行模式

输出目录：

- `output/`

### 9.2 Web 上传模式

输出目录：

- `output_web/`

### 9.3 文件命名

默认会保留原始文件名输出到目标目录。

例如：

- `input/DSC01029.jpg`
- 输出为 `output/DSC01029.jpg` 或 `output_web/DSC01029.jpg`

## 10. EXIF 与字段来源说明

许多文字内容依赖照片 EXIF 信息，例如：

- 相机型号
- 品牌
- 镜头型号
- 焦距
- ISO
- 拍摄时间
- GPS 信息

如果照片没有这些信息，页面上某些字段可能会显示为空或默认占位值。

建议：

- 优先使用原始照片
- 尽量避免提前经过会清除 EXIF 的社交平台或压缩软件

## 12. 常见问题

### 12.1 启动前端时报 `streamlit: command not found`

先确认已经激活环境：

```bash
conda activate semi-utils
```

然后再执行：

```bash
streamlit run web_app.py
```

### 12.2 页面能打开，但处理失败

优先检查：

- 图片格式是否受支持
- 图片是否损坏
- `logos/`、`fonts/` 资源是否存在
- 是否缺少 `exiftool`

另外可以查看日志目录：

- `logs/info.log`
- `logs/error.log`
- `logs/all.log`

### 12.3 没有读取到相机或镜头信息

通常原因有：

- 图片本身不带 EXIF
- 图片已被平台压缩
- 该字段在原始文件中缺失

解决方式：

- 使用原图
- 改用“文件名”或“自定义”作为展示字段

### 12.4 统一尺寸后图片被裁掉了

你当前大概率使用的是：

- `居中裁切`

如果不想裁切，请改成：

- `留白补齐`

### 12.5 统一尺寸后图片看起来变形

你当前大概率使用的是：

- `强制拉伸`

如果不想变形，请改成：

- `留白补齐`
- 或 `居中裁切`

### 12.6 输出图片太大

可以降低输出质量，例如从 `100` 调低到 `90` 或 `85`。

### 12.7 前端处理上传图片后，结果保存到哪里了

上传模式下会保存在：

- `output_web/`

同时页面也支持直接下载。
