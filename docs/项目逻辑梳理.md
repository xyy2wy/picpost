# Semi-Utils 项目逻辑梳理

## 1. 项目是做什么的
Semi-Utils 是一个基于 Python 的照片批处理工具，主要能力包括：
- 批量读取 `input/` 下图片（jpg/jpeg/png）
- 读取 EXIF 信息（机身、镜头、参数、时间、GPS 等）
- 按布局生成水印/边框/模糊背景等效果
- 输出到 `output/`，并尽可能保留原图 EXIF

它的交互方式是命令行菜单，不是 Web 服务或 GUI 应用。

## 2. 整体架构
核心是“配置驱动 + 菜单修改配置 + 处理器链执行图像变换”。

- 入口与状态机：`/Users/xyy/Desktop/semi-utils/main.py`
- 初始化与菜单装配：`/Users/xyy/Desktop/semi-utils/init.py`
- 配置管理：`/Users/xyy/Desktop/semi-utils/entity/config.py`
- 图像上下文容器：`/Users/xyy/Desktop/semi-utils/entity/image_container.py`
- 图像处理器（布局实现）：`/Users/xyy/Desktop/semi-utils/entity/image_processor.py`
- 菜单抽象：`/Users/xyy/Desktop/semi-utils/entity/menu.py`
- 工具函数与 EXIF 读取：`/Users/xyy/Desktop/semi-utils/utils.py`
- 视频生成功能（可选）：`/Users/xyy/Desktop/semi-utils/gen_video.py`

## 3. 主流程（运行一次处理任务）
1. 启动 `main.py`，进入主菜单循环（`state=0`）。
2. 菜单项操作会调用 `Config` 的 setter，实时修改内存中的配置。
3. 每轮循环末尾执行 `config.save()`，把配置写回 `config.yaml`。
4. 输入 `y` 或回车后进入 `state=100`，开始批量处理。
5. `processing()` 扫描输入目录，按当前配置组装 `ProcessorChain`：
   - 可选：`ShadowProcessor`
   - 必选：当前布局对应的 Processor（如 `WatermarkRightLogoProcessor`）
   - 可选：`MarginProcessor`
   - 可选：`PaddingToOriginalRatioProcessor`
6. 使用 `multiprocessing.Pool(5)` 并发处理每张图：
   - `ImageContainer` 加载图片与 EXIF，解析参数
   - 处理链依次变换 `container.watermark_img`
   - 保存到 `output/文件名`（质量来自配置）
7. 全部完成后返回主菜单或退出。

## 4. 配置是如何影响行为的
配置文件：`/Users/xyy/Desktop/semi-utils/config.yaml`

关键映射关系：
- `layout.type`：决定使用哪种主布局处理器（normal、简洁、1:1、背景模糊等）
- `layout.elements.*`：决定四个角落显示哪类信息（Model、Param、Datetime、自定义等）
- `layout.logo_enable/logo_position`：影响自定义 normal 布局的 logo 开关和左右位置
- `global.white_margin.enable/width`：控制是否加白边及宽度
- `global.shadow.enable`：控制是否加阴影（正方形布局会跳过）
- `global.padding_with_original_ratio.enable`：是否按原图宽高比补齐
- `global.focal_length.use_equivalent_focal_length`：参数文字使用真实焦距还是等效焦距
- `base.quality`：输出图片质量

## 5. 关键对象职责

### 5.1 Config
`Config` 封装 `config.yaml` 读写，提供业务语义化接口（如 `enable_shadow()`、`set_layout()`）。
同时管理 logo 缓存与字体加载。

### 5.2 ImageContainer
单张图的上下文对象，职责包括：
- 读取原图与 EXIF
- 解析拍摄参数字符串（焦距/光圈/快门/ISO）
- 处理方向信息（旋转修正）
- 提供统一的属性访问（`get_attribute_str()` 给水印文本用）
- 持有当前处理中间图 `watermark_img`

### 5.3 ProcessorChain 与各 Processor
`ProcessorChain` 是线性管道。每个 Processor 只做一类图像变换并更新容器。
常见处理器：
- `Watermark*Processor`：normal 系列布局（左右文字 + logo）
- `SimpleProcessor`：简洁布局
- `SquareProcessor`：1:1 填充
- `BackgroundBlur*Processor`：背景模糊风格
- `MarginProcessor` / `PureWhiteMarginProcessor`：加边框
- `ShadowProcessor`：加阴影
- `PaddingToOriginalRatioProcessor`：按原比例补齐

## 6. 菜单系统如何工作
菜单组件采用简单组合结构：
- `Menu`：根菜单/容器
- `SubMenu`：有多个可选项，并能根据配置判断“当前激活项”
- `MenuItem`：叶子节点，绑定一个回调函数（通常是 `Config` 的 setter）

`init.py` 负责把“菜单项”和“配置字段”绑定起来。

## 7. EXIF 与外部依赖
- EXIF 读取依赖 `exiftool`（`utils.py` 中通过 subprocess 调用）
- 若系统无 `exiftool`，会尝试使用项目内 `./exiftool/exiftool` 或 Windows 的 `./exiftool/exiftool.exe`
- 视频功能依赖 `ffmpeg`，`gen_video.py` 在 Windows 下支持自动下载

## 8. 并发与日志
- 并发：固定 5 进程（`Pool(5)`）
- 进度：`tqdm` 显示处理进度条
- 日志：`init.py` 初始化 `logs/info.log`、`logs/error.log`、`logs/all.log`

## 9. 二次开发入口（最常用）
1. 新增布局：
   - 在 `entity/image_processor.py` 新建 Processor 子类
   - 在 `init.py` 的 `LAYOUT_ITEMS` 注册
2. 新增可显示字段：
   - 在 `enums/constant.py` 增加字段常量
   - 在 `ImageContainer` 的 `_param_dict` 补充值
   - 在 `init.py` 的 `ITEM_LIST` 注册
3. 新增菜单开关：
   - 在 `Config` 增加读写方法
   - 在 `init.py` 新增对应 `SubMenu/MenuItem`

## 11. 需要留意的实现细节
- `config.save()` 在主循环每轮都会执行，配置改动会持久化。
- 部分布局会显式限制某些开关（例如正方形布局跳过阴影）。
- EXIF 解析依赖外部命令输出格式，遇到特殊机型字段可能要在 `extract_attribute` 里补兼容键。
- 图片对象较多，处理器里频繁创建临时 `Image`，新增功能时要注意内存与 close 时机。

### 第 0 步：准备环境（约 1 分钟）
- 进入项目目录：`cd /Users/xyy/Desktop/semi-utils`
- 安装依赖（首次）：`pip3 install -r requirements.txt`
- 确认目录存在：`input/`、`output/`

### 第 1 步：放入测试图片（约 30 秒）
- 把 1~3 张测试图放到：`/Users/xyy/Desktop/semi-utils/input`
- 建议先用有 EXIF 的 jpg/jpeg，方便看到完整参数文字

### 第 2 步：启动程序（约 30 秒）
- 运行：`python3 /Users/xyy/Desktop/semi-utils/main.py`
- 看到主菜单后，先不要改太多配置

### 第 3 步：先跑一轮默认配置（约 1 分钟）
- 直接输入 `y` 或回车开始处理
- 处理完成后到 `output/` 查看结果
- 先确认流程通了，再做样式调整

### 第 4 步：做两个最常用调整（约 1 分钟）
1. 调整布局：主菜单里进入“布局”，切到你想要的样式（推荐先试 `normal`、`简洁`）
2. 调整白边：进入“更多设置 -> 白色边框”，按需要开关

改完后再次输入 `y` 重新处理，对比输出效果。

### 第 5 步：遇到问题优先检查（约 1 分钟）
1. 没有输出：检查 `input/` 是否有图片、后缀是否为 jpg/jpeg/png
2. 文本不对：检查图片 EXIF 是否完整（无 EXIF 时部分字段会是默认值）
3. logo 不匹配：检查 `config.yaml` 中 `logo.makes` 的 `id` 是否能匹配相机 `Make`
4. 处理报错：看日志文件 `logs/error.log`

2. 第二步再读 `main.py -> init.py -> image_processor.py`
